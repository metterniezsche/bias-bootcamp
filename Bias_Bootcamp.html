Notebook: Sesgos de gnero en actividades de "Team Building"

Este notebook fue realizado como ejercicio de aprendizaje en el marco de mi formacin en Ciencia de Datos. Aunque mi trayectoria proviene de las ciencias sociales, este trabajo combina esa mirada conceptual con las herramientas iniciales que estoy incorporando en anlisis y simulacin de datos.

El punto de partida es un ejemplo pedaggico real: una actividad de integracin, la primera actividad realizada en el marco de un master, donde se propuso formar equipos segn la talla de zapato como dinmica de integracin.

Lo que a primera vista parece un juego inocente, desde una mirada cientfica revela una falla de diseo experimental: la variable usada no es neutral, est correlacionada con el sexo y con la altura, y por tanto reproduce sesgos estructurales en lugar de integrar.

Tal vez el ejercicio buscaba exponer el sesgo y no reproducirlo, pero el diseo tambin comunica su epistemologa. Empezar por lo corporal legitima la idea de que la diferencia fsica es el punto de partida del pensamiento analtico. Y eso es conceptualmente pobre, y un modo desconcertante de comenzar la cursada de un posgrado en ciencia de datos.

Mi respuesta fue usar las herramientas bsicas de Ciencia de Datos simulacin, correlacin y visualizacin para mostrar que el azar era un sesgo determinista. El propsito es mostrar cmo decisiones pedaggicas aparentemente triviales pueden revelar sesgos epistemolgicos profundos.

A lo largo del anlisis se demuestra que este criterio no slo genera clsteres de gnero, sino tambin clsteres de tamao corporal: agrupa mujeres pequeas con mujeres pequeas y hombres grandes con hombres grandes. El resultado es una homogeneidad doble de gnero y de escala que contradice la idea de diversidad funcional y refuerza las jerarquas que la ciencia debera problematizar.

# puede inducir sesgos de gnero y homogeneidad en la formacin de equipos.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Ajustes visuales
plt.rcParams["figure.figsize"] = (7, 4)
plt.rcParams["axes.grid"] = True

# Semilla y tamao de muestra
rng = np.random.default_rng(42)
N = 200

# Variables base
sex = rng.integers(0, 2, size=N)  # 0 = mujer, 1 = hombre
height_cm = np.where(
    sex == 0,
    rng.normal(165, 6, N),   # mujeres
    rng.normal(177, 7, N)    # hombres
)

# ------------------------------
# Talla EU realista en dos pasos
# 1) longitud de pie (cm) como funcin ruidosa de la altura + offset por sexo
a = 0.145                     # pendiente alturapie
b = -0.5                      # intercepto
offset_sex_cm = np.where(sex == 0, 0.0, 0.8)  # hombres ~ +0.8 cm
epsilon = rng.normal(0, 1.0, size=N)          # ruido (ajusta realismo)

foot_len_cm = a * height_cm + b + offset_sex_cm + epsilon  # ~2330 cm

# 2) convertir longitud de pie (cm) a talla EU y redondear a medias tallas
shoe_size = 1.5 * foot_len_cm - 1.5
shoe_size = np.round(shoe_size * 2) / 2
# ------------------------------

# DataFrame final
df = pd.DataFrame({
    "sex": sex,
    "height_cm": height_cm,
    "shoe_size": shoe_size
})

Lectura sobre la simulacin sinttica y sus fundamentos

En esta celda creamos un dataset sinttico de 200 casos.
No son datos reales, sino una simulacin paramtrica: un modelo controlado que imita relaciones poblacionales conocidas para observar sus consecuencias estadsticas sin exponer personas reales.

Cada variable fue generada con una lgica distinta:

  • sex (0 = mujer, 1 = hombre) se asign al azar, slo para crear dos grupos comparables.
  • height_cm se extrajo de distribuciones normales con medias y desviaciones distintas: 165 6 cm para mujeres y 177 7 cm para hombres.
  • shoe_size se deriv de la altura usando una relacin lineal que reproduce el rango real de tallas europeas (3546), con un pequeo offset (+1 para hombres) y ruido aleatorio rng.normal(0, 0.5) para reflejar variacin individual.

Qu estamos haciendo en realidad:
Estamos construyendo un modelo de correlaciones, no un retrato de la realidad.
Sabemos que en la poblacin general existe correlacin entre sexo, altura y talla de zapato; reproducirla aqu nos permite ver cmo un criterio aparentemente trivial (la talla) se comporta como una variable predictiva de gnero.

Por qu esto es vlido y necesario:

  • ticamente seguro: no usa datos personales.
  • Metodolgicamente slido: mantiene proporciones realistas y ruido controlado.
  • Pedaggicamente potente: muestra con claridad cmo nacen los sesgos desde el diseo de variables.

En sntesis: no inventamos personas, modelamos relaciones.
Esa es la diferencia entre manipular datos y hacer ciencia de datos.

Reproducibilidad: usamos semilla fija rng = np.random.default_rng(42) para que cualquiera obtenga los mismos resultados al ejecutar.

Conclusin: usar talla para formar equipos introduce sesgo porque est correlacionada con variables biolgicas/sociales.

def teams_by_shoes(df,K=4):
    d=df.sort_values("shoe_size").copy(); d["team"]=np.arange(len(d))%K; return d
def teams_balanced_by_sex(df,K=4):
    males=df[df.sex==1].sample(frac=1,random_state=42)
    females=df[df.sex==0].sample(frac=1,random_state=24)
    teams=[]
    for i,(_,r) in enumerate(males.iterrows()): teams.append((r.name,i%K))
    for j,(_,r) in enumerate(females.iterrows()): teams.append((r.name,j%K))
    team_map=dict(teams); df=df.copy(); df["team"]=[team_map[i] for i in df.index]; return df
def std_rate_male(d): return d.groupby("team")["sex"].mean().std()

K=4

# body_size_index: ndice compuesto a partir de altura y talla (z-score promedio)

def z(x):

Comparacin de mtodos (gnero por equipo)

Qu medimos y por qu:
std_rate_male es la desviacin estndar de la proporcin de hombres en cada equipo.

  • Primero calculamos, para cada equipo, su proporcin de hombres (entre 0 y 1).
  • Luego medimos qu tan distintas son esas proporciones entre s.
  • Ms bajo = equipos con proporciones de gnero ms parecidas = reparto ms equilibrado.

Mini-ejemplo (intuitivo):
Supongamos 4 equipos y estas proporciones de hombres:

  • Mtodo A [0.10, 0.15, 0.12, 0.13] todas parecidas std_rate_male bajo
  • Mtodo B [0.05, 0.85, 0.60, 0.10] muy distintas std_rate_male alto

Aunque el promedio de hombres sea el mismo, el mtodo B concentra hombres en algunos equipos y los vaca en otros.
Eso es precisamente lo que std_rate_male captura: desbalance entre equipos.

Cmo leer estos resultados:

  • Si por talla tiene std_rate_male ms alto que balanceado, significa que agrupar por talla empeora el equilibrio de gnero entre equipos.
  • Si por tamao corporal (ndice altura+talla) queda igual o peor que por talla, confirma que cualquier criterio corporal tiende a homogeneizar por gnero (unos equipos muy masculinos, otros muy femeninos), en vez de mezclar.

Reglas para leer los resultados:

  • std_rate_male < 0.05 muy equilibrado
  • 0.05 0.10 moderadamente equilibrado
  • > 0.10 desbalance evidente

Eplogo Pedagoga, mtodo y exigencia

Lo que parece un juego inocente es, en realidad, una falla epistemolgica: usar una variable fsica talla de zapato correlacionada con sexo y altura, como si fuera aleatoria. No cuestiono la intencin de integrar, sino la banalizacin del pensamiento cientfico que supone tratar un dato corporal como herramienta pedaggica. Una maestra en ciencia no puede comenzar as: una variable que predice el sexo no es neutra ni ldica, es un error de diseo experimental.

Esta crtica no es moral, es cientfica. Elegir una variable con colinealidad fuerte, asumir independencia donde no la hay y confundir azar con desconocimiento viola principios bsicos de investigacin. Integrar desde el cuerpo no slo es ticamente dbil: es epistemolgicamente improductivo. Agrupa a quienes ya comparten correlaciones previsibles y elimina la posibilidad de innovacin. La verdadera creatividad surge de la diferencia en dilogo, no de la semejanza agrupada. En ciencia de datos, eso equivale a un modelo con varianza nula: no aprende porque todos los datos son iguales.

Pretendo ms. No ms atencin, sino ms rigor, ms profundidad y ms imaginacin pedaggica. Porque la ciencia de datos, bien enseada, puede ser un espacio de creatividad, anlisis y pensamiento libre.
